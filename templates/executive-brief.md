# Template: executive-brief

**Use when:** a fast, high-level answer is needed for a non-lawyer business stakeholder. Short, bullet-heavy, no full IRAC. Default for simple compliance questions or product-team requests.

## Required sections (in this order)

1. **Header** — title, date, jurisdictions, query (1 sentence), template name.
2. **TL;DR** — 3-5 sentences with the bottom-line answer. Action-oriented.
3. **Key assumptions** — only if material; ≤3 bullets. Skip if assumptions do not affect the answer.
4. **Top risks** — bulleted list, ≤5 items. Each item: one sentence stating the risk + one sentence on mitigation.
5. **Recommendation** — clear action item or set of options for the stakeholder.
6. **Sources** — numbered list with title, identifier, URL, retrieval date.

## Tone

Direct, plain-English (or plain Russian). Avoid legalese unless necessary. The reader is a business decision-maker, not a lawyer.

## Length guidance

500-1200 words total. If the answer needs more than this, switch template to `classical-memo` instead.

## Rules

- NO IRAC structure. The reader doesn't need to see Rule/Application/Conclusion separately.
- Plain language: if a legal term is unavoidable, define it on first use in one phrase.
- Bullets > prose where possible.
- If multiple issues from `plan.md` are covered, fold them into TL;DR + Top risks rather than dedicating a section per issue.

## What goes in the warning banner (forced exit / manual review)

Same as classical-memo — yellow callout at the top for any non-approved final status.
